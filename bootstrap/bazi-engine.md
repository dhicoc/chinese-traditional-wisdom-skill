# 八字排盘引擎集成指南

> 本文件指导如何集成 dzcmemory-web/bazi-ziwei-skill 引擎，用于"命"维度的八字四柱排盘与格局分析。

---

## 引擎概览

**项目地址**：https://github.com/dzcmemory-web/bazi-ziwei-skill

**Stars**：520

**技术栈**：TypeScript + lunar-typescript（唯一依赖）

**核心理念**：不让 LLM 自己排盘——使用确定性算法保证命盘准确性，LLM 仅负责解读。

**核心能力**：
- 八字四柱精确排盘（年柱、月柱、日柱、时柱）
- 十神排布（比肩/劫财/食神/伤官/偏财/正财/偏官/正官/偏印/正印）
- 纳音计算
- 神煞安放
- 大运推算（10 步大运，含起运时间）
- 刑冲合会关系分析
- 格局/旺衰/调候分析增强层
- 八字与紫微交叉验证（两套独立体系核对主轴一致性）
- 输出：Markdown 长文 + 水墨风 HTML 海报

**许可**：核心引擎源自 Yiqi 项目（MIT）

**形式**：AI Skill（SKILL.md 格式），兼容 Claude Code / Cursor / Codex / Hermes / OpenClaw

## 集成方式

### 方式一：作为独立 Skill 引用（推荐）

bazi-ziwei-skill 本身就是 SKILL.md 格式的 AI skill，可直接放入 agent 的 skills 目录：

```bash
git clone https://github.com/dzcmemory-web/bazi-ziwei-skill.git
# 将 calculator/ 目录和 SKILL.md 放入技能路径
```

然后在 chinese-traditional-wisdom-skill 的咨询流程中，当需要八字排盘时，按本文件的输入输出规范调用其 calculator 模块。

### 方式二：直接使用 lunar-typescript 库

bazi-ziwei-skill 的排盘核心依赖 lunar-typescript，可独立安装：

```bash
npm install lunar-typescript
```

```typescript
import { Lunar, Solar } from 'lunar-typescript';

// 阳历转八字
const solar = Solar.fromYmdh(1990, 8, 15, 6); // 1990年8月15日卯时
const lunar = solar.getLunar();
const bazi = lunar.getEightChar(); // EightChar 对象

// 四柱输出
bazi.getYear();  // 年柱 "庚午"
bazi.getMonth(); // 月柱 "甲申"
bazi.getDay();   // 日柱 "壬子"
bazi.getTime();  // 时柱 "癸卯"

// 十神
bazi.getDayGanShenZhiShen(); // 日干与地支藏干的关系

// 大运
const yun = bazi.getYun(1); // 1=男, 0=女
yun.getStartYear();    // 起运年
yun.getStartMonth();   // 起运月
yun.getStartDay();     // 起运日
const dayun = yun.getDaYun(0); // 第一步大运
dayun.getGanZhi();     // 大运干支
dayun.getStartYear();  // 大运起始年
dayun.getEndYear();    // 大运结束年
```

### 方式三：MCP 协议调用

bazi-ziwei-skill 支持 MCP 协议，可通过 MCP 工具调用排盘功能，适合 Claude Desktop 等 MCP 兼容环境。

## 排盘输出结构

### 四柱信息

```
年柱：庚午  月柱：甲申  日柱：壬子  时柱：癸卯
```

每柱包含：
- **天干**：甲乙丙丁戊己庚辛壬癸
- **地支**：子丑寅卯辰巳午未申酉戌亥
- **藏干**：地支中暗藏的天干（如寅藏甲丙戊）
- **纳音**：六十甲子纳音（如庚午纳音 = 路旁土）
- **十神**：以日干为我，推演其他干支的十神关系

### 十神体系

| 十神 | 关系 | 含义 |
|------|------|------|
| 比肩 | 同我 | 兄弟、朋友、竞争 |
| 劫财 | 异性同我 | 争夺、破财 |
| 食神 | 我生同性 | 福寿、才艺 |
| 伤官 | 我生异性 | 才华、叛逆 |
| 偏财 | 我克同性 | 偏财、意外之财 |
| 正财 | 我克异性 | 正当收入、妻子 |
| 偏官(七杀) | 克我同性 | 压力、魄力、小人 |
| 正官 | 克我异性 | 名望、事业、丈夫 |
| 偏印(枭神) | 生我同性 | 学业、孤僻 |
| 正印 | 生我异性 | 慈爱、学业、母亲 |

### 大运

大运以月柱为起点，阳男阴女顺行，阴男阳女逆行。每步大运十年。

关键信息：
- **起运时间**：出生日距下一个（或上一个）节气的天数 ÷ 3 = 起运岁数
- **大运干支**：顺行或逆行的六十甲子序列
- **大运十神**：大运天干对日干的十神关系

### 刑冲合会

- **三合局**：申子辰合水局、亥卯未合木局、寅午戌合火局、巳酉丑合金局
- **六合**：子丑合、寅亥合、卯戌合、辰酉合、巳申合、午未合
- **相冲**：子午冲、丑未冲、寅申冲、卯酉冲、辰戌冲、巳亥冲
- **相刑**：寅巳申三刑、丑戌未三刑、子卯刑、辰午酉亥自刑
- **相害**：子未害、丑午害、寅巳害、卯辰害、申亥害、酉戌害

### 格局分析增强层

bazi-ziwei-skill 在原始排盘之上增加格局分析层：

- **格局判断**：正官格、偏官格、正印格、偏印格、食神格、伤官格、正财格、偏财格、从格、化气格等
- **旺衰分析**：日主得令/失令、得地/失地、得势/失势
- **调候分析**：根据月令气候判断是否需要调候
- **用神选取**：扶抑、通关、调候、从势等取用法则

## 八字与紫微交叉验证

bazi-ziwei-skill 的一大特色是八字与紫微斗数的交叉验证：

1. **独立排盘**：八字四柱排盘 + 紫微十二宫排盘（各自独立计算）
2. **主轴核对**：检查两套体系在性格、事业、财运、婚姻等主题上的结论是否一致
3. **互补增强**：八字擅长五行力量分析，紫微擅长宫位星曜分析，两者互补
4. **矛盾处理**：如两套体系结论矛盾，需注明并从多角度解读

交叉验证结果格式：

```json
{
  "theme": "事业方向",
  "bazi_analysis": { "finding": "日主偏弱，正官格，宜公职", "confidence": "high" },
  "ziwei_analysis": { "finding": "官禄宫紫微天府，宜管理岗", "confidence": "high" },
  "cross_verify": "consistent",
  "synthesis": "两套体系均指向管理/公职方向，一致性高"
}
```

## 在咨询中的使用

### 事业决策（场景 B）

1. 排八字四柱，确定日主强弱和格局
2. 查看正官/偏官（事业星）的旺衰和位置
3. 大运判断当前十年事业运势方向
4. 结合紫微官禄宫交叉验证
5. 刑冲合会分析事业中的阻碍和助力

### 婚恋合婚（场景 C）

1. 排双方八字，重点看日柱（日主与配偶宫）
2. 正财/偏财（男看妻星）、正官/偏官（女看夫星）的旺衰
3. 双方八字五行互补性分析
4. 地支合/冲/刑/害关系
5. 结合紫微夫妻宫交叉验证

### 健康养生（场景 A）

1. 排八字，确定五行偏颇（如水旺火弱）
2. 五行与脏腑对应（参考 reference-tcm.md）
3. 大运流年中五行力量的变化对健康的影响
4. 天克地冲年份的健康预警
5. 结合紫微疾厄宫交叉验证

## 与紫微斗数的配合

bazi-ziwei-skill 同时支持八字和紫微排盘，与 bootstrap/ziwei-engine.md 的关系：

| 维度 | 八字擅长 | 紫微擅长 |
|------|---------|---------|
| 性格分析 | 五行力量、十神心性 | 主星特质、宫位组合 |
| 事业 | 格局+大运+十神 | 官禄宫+大限+四化 |
| 财运 | 正偏财旺衰+大运 | 财帛宫+化禄/化忌 |
| 婚姻 | 日柱+夫妻星+合婚 | 夫妻宫+福德宫 |
| 健康 | 五行偏颇+流年冲克 | 疾厄宫+煞星会照 |

建议：事业和财运以八字大运为主、紫微宫位为辅；婚姻和健康以紫微宫位为主、八字五行为辅。

## 注意事项

- 八字排盘需要准确的出生时辰，时辰错误会导致四柱全错
- 真太阳时校正：如知道出生地经纬度，应进行地方时校正
- 晚子时（23:00-24:00）属于次日，需特别注意
- 闰月对八字的影响主要体现在月柱的节气界定，非农历月份
- bazi-ziwei-skill 使用 lunar-typescript 库，排盘算法已验证，但格局解读仍需人工判断
- 所有命理分析必须遵守 RULES.md 中的 disclaimer 和积极导向原则
